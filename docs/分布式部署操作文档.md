# 运单系统双服务器分布式部署说明（验收版）

> 如需满足“停任意一台服务器后另一台仍可办理核心业务”，请配合阅读《双机高可用最小改造方案》：`docs/双机高可用最小改造方案.md`。

> 分布式故障切换与双节点恢复的工作机制说明见：`docs/分布式工作原理说明.md`。

## 1. 文档目的

本文档用于把当前项目稳定部署到两台 Linux 服务器，并满足“可访问、可切换、可验收”的交付要求。

适用资源：

- 控制台：Windows（本机）
- 服务器1：192.168.5.81（入口：122.224.127.36:2281）
- 服务器2：192.168.5.82（入口：122.224.127.36:2282）

公网访问地址：

- http://122.224.127.36:18080

## 2. 部署拓扑

服务器1（业务入口）：

- gateway（Nginx）
- web（前端）
- api-1（后端实例1）
- mysql（本机依赖）
- redis（本机依赖）
- rabbitmq（本机依赖）

服务器2（共享依赖 + 后端）：

- api-2（后端实例2）
- mysql
- redis
- rabbitmq

端口规划：

- 服务器1：18080(gateway), 3001(api-1)
- 服务器2：3000(api-2), 13306(mysql), 16379(redis), 5673/15673(rabbitmq)

## 3. 前置检查

在本机 PowerShell 执行：

```powershell
ssh -V
scp -V
Test-NetConnection 122.224.127.36 -Port 2281
Test-NetConnection 122.224.127.36 -Port 2282
```

在两台 Linux 分别执行（第一次登录后）：

```bash
docker -v
docker compose version
```

## 4. 部署文件说明

项目中已提供分布式部署文件：

- deploy/distributed/server1/docker-compose.yml
- deploy/distributed/server1/gateway.conf
- deploy/distributed/server2/docker-compose.yml

本次部署目录统一使用：

- ~/waybill-dist

## 5. 环境变量准备（关键）

在项目根目录准备 `.env`（本地已有可复用）。至少包含：

```env
DEV_LOGIN_ENABLED=1
DEV_LOGIN_PASSWORD=123456
AUTH_SESSION_SECRET=waybill-auth-session-secret
GOOGLE_CLIENT_ID=你的GoogleClientId
GOOGLE_ADMIN_EMAILS=admin@example.com
GOOGLE_CARRIER_EMAILS=carrier@example.com
GOOGLE_SHIPPER_EMAILS=shipper@example.com
```

说明：

- 不建议把服务器 SSH 密码写入仓库文件。
- 线上如需开启 Google 登录，`GOOGLE_CLIENT_ID` 必填。

## 6. 步骤A：部署服务器2（api-2/mysql/redis/rabbitmq）

以下命令在本机 PowerShell 执行。

1. 初始化远端目录

```powershell
ssh -p 2282 xlab@122.224.127.36 "rm -rf ~/waybill-dist; mkdir -p ~/waybill-dist/apps ~/waybill-dist/db"
```

2. 上传代码和配置

```powershell
scp -P 2282 package.json tsconfig.base.json .env xlab@122.224.127.36:~/waybill-dist/
scp -P 2282 -r apps/api xlab@122.224.127.36:~/waybill-dist/apps/
scp -P 2282 -r db/init xlab@122.224.127.36:~/waybill-dist/db/
scp -P 2282 deploy/distributed/server2/docker-compose.yml xlab@122.224.127.36:~/waybill-dist/docker-compose.yml
```

3. 预拉取镜像并启动

```powershell
ssh -p 2282 xlab@122.224.127.36 "cd ~/waybill-dist; docker pull docker.m.daocloud.io/library/node:22-alpine; docker tag docker.m.daocloud.io/library/node:22-alpine node:22-alpine; docker pull docker.m.daocloud.io/library/mysql:8.4; docker tag docker.m.daocloud.io/library/mysql:8.4 mysql:8.4; docker pull docker.m.daocloud.io/library/redis:7-alpine; docker tag docker.m.daocloud.io/library/redis:7-alpine redis:7-alpine; docker pull docker.m.daocloud.io/library/rabbitmq:3.13-management; docker tag docker.m.daocloud.io/library/rabbitmq:3.13-management rabbitmq:3.13-management; docker compose up -d --build"
```

4. 查看状态

```powershell
ssh -p 2282 xlab@122.224.127.36 "cd ~/waybill-dist && docker compose ps"
```

## 8. 验收验证步骤（必须执行）

1. 网关健康检查（服务器1）

```powershell
ssh -p 2281 xlab@122.224.127.36 "curl -s -o /dev/null -w '%{http_code}\n' http://127.0.0.1:18080/health"
```

期望：`200`

2. 服务器1到服务器2依赖连通（可选检查）

```powershell
ssh -p 2281 xlab@122.224.127.36 "nc -zv 192.168.5.82 13306; nc -zv 192.168.5.82 16379; nc -zv 192.168.5.82 5673"
```

期望：三个端口均 `succeeded`

说明：

- 阶段 A 已将服务器1改造为本机依赖优先，即使此项失败，服务器1仍可单机继续提供核心业务。

3. HA 实例返回

```powershell
ssh -p 2281 xlab@122.224.127.36 "curl -s http://127.0.0.1:18080/api/ha/instance"
```

期望：返回 `api-1` 或 `api-2`

4. 故障切换验证

```powershell
ssh -p 2281 xlab@122.224.127.36 "cd ~/waybill-dist && docker compose stop api-1 >/dev/null 2>&1; sleep 2; curl -s http://127.0.0.1:18080/api/ha/instance; docker compose start api-1 >/dev/null 2>&1; sleep 2; curl -s http://127.0.0.1:18080/api/ha/instance"
```

期望：停止后返回 `api-2`，恢复后返回 `api-1` 或 `api-2`

5. 公网回测

```powershell
Test-NetConnection 122.224.127.36 -Port 18080
Invoke-WebRequest http://122.224.127.36:18080/health -UseBasicParsing
Invoke-RestMethod http://122.224.127.36:18080/api/auth/config
```

期望：

- 端口连通 `TcpTestSucceeded=True`
- `health` 返回 200
- 如已配置 Google，`googleEnabled=true`

## 9. 日常更新流程

当代码有更新时，按下面顺序执行：

1. 先更新服务器2（api-2 + 依赖）
2. 再更新服务器1（api-1 + web + gateway）
3. 最后执行第 8 章验收验证步骤

更新命令模板：

```powershell
ssh -p 2282 xlab@122.224.127.36 "cd ~/waybill-dist && docker compose up -d --build --force-recreate"
ssh -p 2281 xlab@122.224.127.36 "cd ~/waybill-dist && docker compose up -d --build --force-recreate"
```

## 10. 回滚方案

如果某次更新后异常：

1. 在两台服务器执行 `docker compose logs` 查看报错
2. 回退到上一版部署目录（建议保留 `~/waybill-dist-prev`）
3. 重新 `docker compose up -d`

建议在每次发布前先备份：

```powershell
ssh -p 2281 xlab@122.224.127.36 "rm -rf ~/waybill-dist-prev; cp -r ~/waybill-dist ~/waybill-dist-prev"
ssh -p 2282 xlab@122.224.127.36 "rm -rf ~/waybill-dist-prev; cp -r ~/waybill-dist ~/waybill-dist-prev"
```

## 11. 常见问题

1. Docker Hub 拉取慢或失败

- 先 pull `docker.m.daocloud.io` 镜像并 tag 回官方镜像名。

2. 页面提示“Google 登录未启用”

- 确认两台服务器 `~/waybill-dist/.env` 中 `GOOGLE_CLIENT_ID` 已配置。
- 执行：

```powershell
ssh -p 2281 xlab@122.224.127.36 "cd ~/waybill-dist && docker compose up -d --force-recreate api-1"
ssh -p 2282 xlab@122.224.127.36 "cd ~/waybill-dist && docker compose up -d --force-recreate api-2"
```

3. 网关返回 502

- 检查 `gateway.conf` upstream 地址和端口。
- 检查 `api-1/api-2` 是否健康并对外监听。

4. 端口冲突

- 调整 compose 端口映射，避免占用系统已有服务端口。
