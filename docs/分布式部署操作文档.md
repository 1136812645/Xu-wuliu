# 运单系统分布式部署操作文档

## 1. 目标与范围

本文档用于记录本项目在以下资源条件下的可执行部署方案：

- 本机 Windows 作为发布与运维控制台
- Linux 服务器 1：192.168.5.81（SSH 入口：122.224.127.36:2281）
- Linux 服务器 2：192.168.5.82（SSH 入口：122.224.127.36:2282）

部署目标：

- Web 分布式部署
- App 分布式部署
- RabbitMQ、MySQL、Redis 作为共享后端依赖（当前落在服务器 2）
- 支持横向扩展与故障切换验证

---

## 2. 架构与角色分工

### 2.1 服务器角色

- 服务器 1（192.168.5.81）
  - gateway（Nginx）
  - web
  - api-1

- 服务器 2（192.168.5.82）
  - api-2
  - mysql
  - redis
  - rabbitmq

### 2.2 当前端口规划

- 服务器 1
  - gateway 对外：18080 -> 80
  - api-1 对外：3001 -> 3000

- 服务器 2
  - api-2 对外：3000 -> 3000
  - mysql 对外：13306 -> 3306
  - redis 对外：16379 -> 6379
  - rabbitmq AMQP：5673 -> 5672
  - rabbitmq 管理台：15673 -> 15672

说明：

- 端口规划采用避冲突策略，避免与已有服务占用默认端口（如 3306、6379、5672、8080、3000）冲突。

## 5. 本机部署（单机可用验证）

### 5.1 解决本机 MySQL 3306 冲突

项目根 docker-compose.yml 已改为可配置 MySQL 映射端口：

- MYSQL_PUBLISHED_PORT 未设置时默认 3306
- Windows 本机已占用 3306 时使用 13306

### 5.2 启动命令

在项目根目录执行：

```powershell
$env:MYSQL_PUBLISHED_PORT="13306"
docker compose up -d
```

### 5.3 状态检查

```powershell
$env:MYSQL_PUBLISHED_PORT="13306"
docker compose ps
```

### 5.4 本机访问验证

- 健康检查：

```powershell
Invoke-WebRequest http://127.0.0.1:8080/health -UseBasicParsing
```

## 6. 双机分布式部署步骤

以下步骤在本机执行。

### 6.1 部署到服务器 1（gateway/web/api-1）

1) 创建目录

```powershell
ssh -p 2281 xlab@122.224.127.36 "mkdir -p ~/waybill-dist/apps ~/waybill-dist/db"
```

2) 上传构建上下文

```powershell
scp -P 2281 package.json tsconfig.base.json xlab@122.224.127.36:~/waybill-dist/
scp -P 2281 -r apps/api apps/web db/init xlab@122.224.127.36:~/waybill-dist/
scp -P 2281 deploy/distributed/server1/docker-compose.yml deploy/distributed/server1/gateway.conf xlab@122.224.127.36:~/waybill-dist/
```

3) 拉取基础镜像并启动

```powershell
ssh -p 2281 xlab@122.224.127.36 "cd ~/waybill-dist && docker pull docker.m.daocloud.io/library/node:22-alpine && docker tag docker.m.daocloud.io/library/node:22-alpine node:22-alpine && docker pull docker.m.daocloud.io/library/nginx:1.27-alpine && docker tag docker.m.daocloud.io/library/nginx:1.27-alpine nginx:1.27-alpine && docker compose up -d --build"
```

4) 查看状态

```powershell
ssh -p 2281 xlab@122.224.127.36 "cd ~/waybill-dist && docker compose ps"
```

### 6.2 部署到服务器 2（api-2/mysql/redis/rabbitmq）

1) 创建目录

## 8. 公网访问与端口映射说明

当前应用公网访问目标为：

- http://122.224.127.36:18080

若公网无法访问，但服务器本机访问 127.0.0.1:18080 返回 200，说明应用正常，问题在公网入口层（NAT 或安全组）。

### 8.1 已排查项

- 服务器监听正常（18080 已监听）
- 服务器本机健康检查正常（200）
- UFW 非阻塞（状态 inactive）

### 8.2 需要在公网网关新增映射

- 协议：TCP
- 外网：122.224.127.36:18080
- 内网：192.168.5.81:18080

完成映射后，从本机回测：

```powershell
Test-NetConnection 122.224.127.36 -Port 18080
Invoke-WebRequest http://122.224.127.36:18080/health -UseBasicParsing
```

## 9. 常见问题与处理

### 9.1 Docker Hub 拉取超时

现象：拉取 node:22-alpine 失败。

处理：

- 使用可访问镜像源拉取并打标准标签，例如：

```bash
docker pull docker.m.daocloud.io/library/node:22-alpine
docker tag docker.m.daocloud.io/library/node:22-alpine node:22-alpine
```

### 9.2 端口冲突

现象：Bind for 0.0.0.0:<port> failed。

处理：

- 修改发布端口，避免与现有服务冲突。
- 本次已采用：
  - gateway: 18080
  - api-1: 3001
  - mysql: 13306
  - redis: 16379
  - rabbitmq: 5673/15673

### 9.3 网关返回 502

排查顺序：

1) 网关 upstream 指向端口是否与 API 实际发布端口一致
2) API 容器是否已发布宿主机端口
3) 从网关容器内是否可直连两台 API

## 11. 后续优化建议

- 若需 RabbitMQ 真正分布式高可用，建议升级为 3 节点集群（奇数节点仲裁）
