# 自测用例：异常与边界覆盖清单

> 标注：自测用例
> 版本：v1.0
> 日期：2026-07-07
> 范围：运单创建、签收/回单幂等、车辆档案并发修改、MQ 可靠性、证件档案校验

## 1. 目标与判定标准

- 目标：覆盖业务边界、幂等边界、并发边界、MQ 异常、档案边界。
- 判定：每条用例都包含前置条件、输入、步骤、预期结果；出现 500 或脏数据视为失败。

## 2. 执行环境

- API: http://127.0.0.1:3000
- DB: waybill_admin
- 建议并发工具：JMeter / Postman Runner / k6（任选其一）
- 建议辅助查询：MySQL 客户端 + /api/mq/status

## 4. 并发修改同一车辆（C-03）建议脚本化步骤

1. 准备同一 vehicleId（建议 vehicle-3）。
2. 线程组 A：并发 20 次 PUT /api/archives/vehicles/:id，truckType=9.6M，maxWeightKg=18000。
3. 线程组 B：并发 20 次 PUT /api/archives/vehicles/:id，truckType=6.8M，maxWeightKg=10000。
4. 全量结束后 GET /api/archives/vehicles/:id。
5. 校验：
   - 返回非 500；
   - 结果字段组合必须来自某一次完整写入（不允许 truckType 来自 A、maxWeightKg 来自 B 的脏组合）；
   - 记录最终值与最后成功请求（用于判定是否存在丢更新风险）。

## 5. MQ 异常执行要点

- 消息丢失：停 MQ 后创建运单，检查 /api/mq/status 的 publishFailed 与 outboxSize。
- 重复消息：重复发送同 eventId，检查 inbox_event 是否仅首次插入。
- 死信消息：投递非法体或制造消费异常，验证 DLQ 积压与重试计数。
