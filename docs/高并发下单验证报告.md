# 高并发下单验证报告（2026-07-07）

## 1. 验证目标

验证在 50~200 线程并发创建运单时，系统是否满足以下要求：

- 数据库运单数量 = 成功请求数量，无多生成、无丢失；
- 运费明细、车辆占用数据无错乱、无重复扣款；
- 分布式锁拦截返回正常业务错误，不抛出 500 异常。

## 2. 验证环境

- API：`@waybill/api`，DB 模式，`http://127.0.0.1:3000`
- 数据库：本机 MySQL `waybill_admin`
- Redis：本地 Redis shim，端口 `6380`
- 测试车辆：`vehicle-3`

说明：`vehicle-3` 为本次验证新增的空闲车辆，避免与历史数据中的活跃运单冲突。

## 3. 验证脚本

执行脚本：

- `node scripts/verify-concurrent-waybill.mjs`

脚本逻辑：

- 先清理测试车辆相关活跃运单；
- 分别执行 50 线程与 200 线程并发创建运单；
- 校验 HTTP 返回码分布；
- 核对数据库中的运单总数、车辆活跃占用数、费用明细数和创建日志数；
- 对成功创建的运单执行签收和回单上传，避免残留活跃占车。

## 5. 结论

本次验证通过。

- 高并发下单时，仅有 1 个请求成功创建运单，其余请求被分布式锁正常拦截为 409；
- 未出现 500 异常；
- 运单总数与成功请求数一致，未见多生成或丢失；
- 运费明细记录正常，费用项数量与类型数符合预期；
- 车辆占用在收尾后恢复为空闲，未观察到重复占车或数据错乱。
