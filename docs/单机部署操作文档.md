# 运单系统单机部署操作文档（验收版）

## 1. 适用范围

本文档用于在一台机器上完成完整部署与验收，适用于：

- 本地开发机（Windows / Linux / macOS）
- 单机演示环境
- 验收前的快速回归环境

部署结果：

- 网关入口：http://127.0.0.1:8080
- 管理后台：http://127.0.0.1:8080
- API 由网关转发到 api-1 / api-2
- RabbitMQ 管理台：http://127.0.0.1:15672

## 2. 单机架构

单机通过 Docker Compose 启动以下服务：

- gateway（Nginx）
- web（前端）
- api-1、api-2（双实例后端）
- mysql、redis、rabbitmq（共享基础设施）

说明：

- 虽是单机，但保留双 API 实例，可用于模拟分布式工作与故障切换。

## 3. 前置条件

### 3.1 软件要求

- Docker Desktop / Docker Engine 可用
- Docker Compose v2 可用
- Node.js >= 22（仅在本地 npm 检查时需要）

### 3.2 快速自检命令

```powershell
docker -v
docker compose version
```

## 4. 环境变量准备

在项目根目录创建 `.env`（可由 `.env.example` 复制）：

```powershell
Copy-Item .env.example .env -Force
```

建议最小配置如下：

```env
# 本机若 3306 被占用可改为 13306
MYSQL_PUBLISHED_PORT=3306

# 登录相关
DEV_LOGIN_ENABLED=1
DEV_LOGIN_PASSWORD=123456
AUTH_SESSION_SECRET=waybill-auth-session-secret

# Google 登录（可选，不配置则前端提示未启用）
GOOGLE_CLIENT_ID=
GOOGLE_ADMIN_EMAILS=admin@example.com
GOOGLE_CARRIER_EMAILS=carrier@example.com
GOOGLE_SHIPPER_EMAILS=shipper@example.com
```

说明：

- `MYSQL_PUBLISHED_PORT` 只影响宿主机映射端口，不影响容器内部 3306。
- 若需要 Google 登录，必须填写 `GOOGLE_CLIENT_ID`。

## 5. 一键启动

在项目根目录执行：

```powershell
docker compose up -d --build
```

查看状态：

```powershell
docker compose ps
```

期望服务状态：

- gateway、web、api-1、api-2、mysql、redis、rabbitmq 均为 Up

## 6. 部署后验收

### 6.1 健康检查

```powershell
Invoke-WebRequest http://127.0.0.1:8080/health -UseBasicParsing
```

期望：`StatusCode = 200`

### 6.2 API 实例探针

```powershell
Invoke-RestMethod http://127.0.0.1:8080/api/ha/instance
```

期望：返回 `instanceId=api-1` 或 `api-2`

### 6.3 登录与业务接口（验收口径）

```powershell
$body = @{ email='1806909748@qq.com'; password='123456'; role='ADMIN' } | ConvertTo-Json
$login = Invoke-RestMethod -Uri 'http://127.0.0.1:8080/api/auth/dev-login' -Method Post -ContentType 'application/json' -Body $body
$token = $login.token
Invoke-RestMethod -Uri 'http://127.0.0.1:8080/api/dashboard' -Headers @{ Authorization = "Bearer $token" }
Invoke-RestMethod -Uri 'http://127.0.0.1:8080/api/waybills' -Headers @{ Authorization = "Bearer $token" }
```

期望：登录成功、dashboard 与 waybills 均正常返回。

### 6.4 双实例分发验证

```powershell
$counts = @{}
1..20 | ForEach-Object {
  $r = Invoke-RestMethod -Uri 'http://127.0.0.1:8080/api/ha/instance' -Method Get
  $id = [string]$r.instanceId
  if (-not $counts.ContainsKey($id)) { $counts[$id] = 0 }
  $counts[$id]++
}
$counts.GetEnumerator() | Sort-Object Name
```

期望：统计结果中出现 `api-1` 与 `api-2`。

## 7. 单机故障演练（可选）

### 7.1 停一个 API 实例

```powershell
docker compose stop api-1
Invoke-RestMethod http://127.0.0.1:8080/api/ha/instance
```

期望：返回 `api-2`，系统仍可登录与查询。

### 7.2 恢复实例

```powershell
docker compose start api-1
```

## 8. 常见问题排查

### 8.1 3306 端口冲突

现象：mysql 启动失败，提示端口占用。

处理：

1. 修改 `.env`：`MYSQL_PUBLISHED_PORT=13306`
2. 重新启动：

```powershell
docker compose down
docker compose up -d --build
```

### 8.2 Google 登录未启用

现象：登录页提示“Google 登录未启用”。

处理：

1. 在 `.env` 中填写 `GOOGLE_CLIENT_ID`
2. 重建 API：

```powershell
docker compose up -d --build --force-recreate api-1 api-2 gateway
```

### 8.3 导入速度慢

建议：

1. 前端分批大小设置为 800 或 1000
2. 确认机器资源充足（磁盘、CPU、内存）
3. 查看 API 日志排查慢批次：

```powershell
docker compose logs --tail=200 api-1 api-2
```

## 9. 运行维护命令

查看状态：

```powershell
docker compose ps
```

查看日志：

```powershell
docker compose logs --tail=200 gateway web api-1 api-2 mysql redis rabbitmq
```

重建全部服务：

```powershell
docker compose up -d --build --force-recreate
```

停止并清理：

```powershell
docker compose down
```

停止并清理（含数据卷，慎用）：

```powershell
docker compose down -v
```

## 10. 结论

按本文档执行后，单机环境可稳定提供：

1. 可访问入口
2. 可登录与核心业务可用
3. 双实例分发能力
4. 单实例故障下的连续性演练能力
