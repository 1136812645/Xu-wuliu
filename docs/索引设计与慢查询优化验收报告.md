### B. 报表多表关联与跨分表统计索引优化方案（补充说明）

1. 对账/报表主路径采用“明细入库 + 日聚合表”分层：
    - 明细层：`waybill_YYYYMM_N` + `waybill_fee_detail`
    - 报表层：`waybill_report_daily`

2. 多表关联优化要点：
    - 费用对账按 `waybill_no` 关联时，使用 `waybill_fee_detail(waybill_no, fee_type)` 复合索引，避免全表扫；
    - 运单分页固定 `ORDER BY created_at DESC, id DESC`，使用 `waybill_xxx(created_at, id)` 游标索引，避免 filesort。

3. 跨分表统计优化要点：
    - 在线查询侧：按月路由后仅扫描当月分片，避免跨月全库 UNION；
    - 经营报表侧：通过 `waybill_report_daily` 预聚合承接“运单量/营收/毛利”查询，减少跨分片实时聚合压力；
    - 索引布局：`uk_waybill_report_daily(report_date, shipper_id, carrier_id)` + `idx_*_date` 支撑日期范围与维度下钻。

4. 慢 SQL 常态化治理：
    - 保留 `db/init/05_slow_query_validation.sql` 作为 EXPLAIN 基线；
    - 结合 `scripts/verify-index-slowquery-acceptance.mjs` 形成“索引覆盖 + 性能阈值”自动验收。
