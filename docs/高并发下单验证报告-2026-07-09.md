# 高并发下单验证报告（2026-07-09）

## 1. 验证目标

本次验证针对高并发创建运单场景，检查以下三项硬性指标：

1. 数据库运单数量增量 = 成功创建请求数量。
2. 运费明细与车辆占用无错乱，无重复扣款。
3. 并发锁拦截行为正常，无 500 错误。

## 3. 验证方法

- 脚本：scripts/verify-concurrent-waybill.mjs
- 并发档位：50、100、200
- 每档执行流程：
  1. 预清理测试车辆活跃单（签收/回单）。
  2. 同步发起 N 个并发建单请求（同 vehicleId，不同幂等键）。
  3. 统计响应分布（201/409/400/500/网络错误）。
  4. 校验数据库增量、活跃占用、费用明细、CREATE 操作日志一致性。
  5. 对成功单据执行收尾流转，避免污染下一档。
