# 司机、车辆证件过期自动预警验收报告（2026-07-09）

## 1. 背景与缺口

现状问题：证件预警页以车辆为主行，司机预警仅在“车辆已绑定司机”时可见，未绑定车辆的司机证件预警会被遗漏。

本次修复：

- 前端补齐“司机独立预警行”，未绑定车辆的司机也会进入预警列表。
- 保持原有联合判定逻辑：车辆道路证与司机驾驶证任一命中过期/临期，整行标红。
- 保持非法日期展示为“无效/非法日期”，不抛异常、不阻塞页面。

对应代码：

- apps/web/src/App.tsx

## 2. 验收标准与结果

### 标准1：过期/临期数据统一标红提醒

结果：通过。

说明：

- 预警行状态判定仍为道路证与驾驶证联合判定。
- 任一状态为 EXPIRED 或 EXPIRING_SOON 即触发行级红色高亮。
- 新增司机独立行后，未绑定车辆的司机也能进入同一红色告警体系。

### 标准2：支持筛选“证件过期”清单

结果：通过。

说明：

- 预警页保留“仅证件过期”筛选。
- 筛选条件覆盖道路证或驾驶证任一为 EXPIRED。
- 司机独立行纳入同一筛选逻辑。

接口侧复核（已执行）：

- 创建一条已过期司机证件数据后，/api/warnings 返回该司机为 EXPIRED。
- 采样输出：
  - createdExpiredDriverStatus: EXPIRED
  - createdExpiredDaysRemaining: -1285

### 标准3：非法日期不产生异常报错、不阻塞页面

结果：通过。

说明：

- 后端逻辑对非法日期按 INVALID 分类，不在页面侧做 Date 强解析抛错。
- 前端状态文案对 INVALID 显示“非法日期/Invalid date”。
- 自动化测试通过：logic.test 中“driver/vehicle warning + invalid date”用例通过。
